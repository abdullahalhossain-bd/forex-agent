# Fixes applied — 2026-07-15

এই ফাইলে সততার সাথে লেখা আছে কোনটা আসলেই "কোড বাগ" ছিল আর fix হয়েছে, আর কোনটা
কোড দিয়ে fix করা যায় না (external service / infra / market reality)। কেউ একটা
বললে "সব ফিক্সড" — সেটা মিথ্যা প্রমিজ হবে, তাই এখানে দুই ভাগে ভাগ করলাম।

## ✅ আসল কোড বাগ — Fixed

1. **`ImportError: cannot import name 'train_all' from 'ml.model_trainer'`**
   `train_all` আগে শুধু `ModelTrainer` ক্লাসের method ছিল, module-level function
   ছিল না। এখন `ml/model_trainer.py`-তে একটা module-level `train_all()` wrapper
   যোগ করা হয়েছে — config.SYMBOLS এর সব পেয়ার train করে। এখন এটা কাজ করবে:
   ```python
   from ml.model_trainer import train_all
   train_all()
   ```

2. **`No module named ml.ensemble_train`**
   এই module টা repo-তে কখনোই ছিল না। নতুন `ml/ensemble_train.py` তৈরি করা
   হয়েছে (`python -m ml.ensemble_train`)। এটা সব পেয়ারের base model retrain করে
   এবং বর্তমান ensemble weights + live performance stats print করে।
   **সততার নোট:** এই কোডবেসে সত্যিকারের "batch ensemble trainer" বলে কিছু নেই —
   ensemble weight আসলে live trade outcome থেকে আপডেট হয় (`ensemble_store.py`),
   কোনো offline training script থেকে না। তাই নতুন script টা বাস্তব pipeline-কেই
   প্রতিফলিত করে, fake কিছু বানায়নি।

3. **`run_backtest.py: error: unrecognized arguments: --days 60 --report`**
   `--days` (day count কে bars-এ convert করে, tf অনুযায়ী) আর `--report`
   (compatibility no-op — report এমনিতেই সবসময় print হয়) flag যোগ করা হয়েছে।

4. **SafePickle: "No metadata file... cannot verify integrity" (12×)**
   ৬টা পেয়ারের সব existing `.pkl` model file-এর জন্য `.meta` sidecar ছিল না।
   `scripts/regenerate_pickle_metadata.py` চালিয়ে সব ২৬টা model file-এর জন্য
   metadata regenerate করা হয়েছে (`regenerated: true` ফ্ল্যাগসহ, যাতে কেউ এটাকে
   original training-time hash বলে ভুল না করে)।

## ⚠️ কোড দিয়ে সত্যিকারভাবে fix করা যায় না — এগুলো এখনো open

1. **MT5 Login failed (-10005, IPC timeout) / Health check failures**
   এটা আপনার লোকাল Windows মেশিনে MetaTrader5 টার্মিনাল আসলেই চালু ও লগইন করা
   আছে কিনা তার উপর নির্ভর করে। কোড থেকে এটা fix করা সম্ভব না — টার্মিনাল খোলা,
   broker-এ লগইন করা, এবং "Allow algo trading" enabled আছে কিনা চেক করতে হবে।

2. **Groq RateLimitError (daily token quota 100000 শেষ)**
   এটা আপনার Groq account-এর quota সীমা, কোড বাগ না। Options: অন্য key যোগ করা,
   Groq Dev Tier-এ upgrade করা, অথবা existing Gemini/Cerebras fallback ব্যবহার
   চলতে দেওয়া (log অনুযায়ী fallback ইতিমধ্যে কাজ করছে)।

3. **Forex Factory 403 Forbidden / Myfxbook parser 0 pairs**
   এগুলো third-party ওয়েবসাইটের bot-detection / HTML পরিবর্তনের কারণে। এই
   scraping bypass করার কোড লেখা আমি এড়িয়ে গেছি — সেটা ওই সাইটগুলোর ToS ভাঙার
   দিকে যায়। System ইতিমধ্যে graceful fallback (hardcoded calendar data /
   synthetic sentiment) ব্যবহার করছে, যেটা এখনকার সবচেয়ে নিরাপদ সমাধান। বিকল্প
   পথ: কোনো official/paid economic-calendar API (যেমন Forex Factory-এর official
   API partner, বা TradingEconomics) ব্যবহার করা।

4. **6 পেয়ারে ML model NOT_READY (USDCHF, NZDUSD, EURJPY, EURGBP, EURCHF, EURAUD)**
   এই পেয়ারগুলোর জন্য কখনো train করা হয়নি — এটা bug না, শুধু এখনো training run
   করা হয়নি। এখন train করতে:
   ```bash
   python scripts/train_models.py --pair USDCHF --tf 15m
   # ... বাকি ৫টা পেয়ারের জন্যও একইভাবে
   ```
   এর জন্য প্রতিটা পেয়ারের যথেষ্ট historical feature data থাকা লাগবে
   (feature_store/dataset_builder) — সেটা আমার কাছে নেই তাই আমি নিজে থেকে
   train করে দিতে পারিনি।

5. **Backtest statistical validation FAIL (score 20/100), GBPUSD -52% drawdown**
   এটা strategy-র performance issue, কোনো software bug না। কোড পরিবর্তন করে
   কৃত্রিমভাবে "profitable" দেখানো সম্ভব কিন্তু সেটা misleading এবং বাস্তবে
   টাকা হারাবে। Walk-forward efficiency 0% মানে strategy in-sample এ যা শিখছে
   out-of-sample এ কাজ করছে না (overfitting)। এটা ঠিক করতে হলে entry logic /
   feature set / risk rules নিয়ে বসে কাজ করতে হবে — এটা একটা আলাদা, বড় কাজ,
   চাইলে সেটা নিয়ে আলাদাভাবে বসতে পারি।
